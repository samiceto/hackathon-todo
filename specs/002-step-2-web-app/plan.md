# Implementation Plan: Full-Stack Web Application

**Branch**: `002-step-2-web-app` | **Date**: 2026-01-08 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/002-step-2-web-app/spec.md`

**Note**: This plan transforms the Step 1 console application into a multi-user web application with authentication, database persistence, and RESTful API.

## Summary

Transform the console todo application into a full-stack web application supporting multiple users with secure authentication and persistent storage. The application will use Next.js 16+ for the frontend, Python FastAPI for the backend API, and Neon PostgreSQL for data storage. Better Auth will handle user authentication with JWT tokens securing all API endpoints. Each user will have isolated access to their own tasks, with complete CRUD operations available through a responsive web interface that works on both desktop and mobile devices.

## Technical Context

**Language/Version**:
- Backend: Python 3.13+ (matching Step 1)
- Frontend: TypeScript 5+ with Next.js 16+

**Primary Dependencies**:
- Backend: FastAPI, SQLModel, PyJWT, python-jose, passlib, uvicorn
- Frontend: Next.js 16+, React 18+, Better Auth, Tailwind CSS 3+, axios/fetch API client

**Storage**: Neon Serverless PostgreSQL (credentials provided by user)

**Testing**:
- Backend: pytest, pytest-asyncio, httpx (for FastAPI testing)
- Frontend: Jest, React Testing Library, Playwright (for E2E)

**Target Platform**:
- Development: localhost (backend: port 8000, frontend: port 3000)
- Production: Web browsers (Chrome, Firefox, Safari, mobile browsers)

**Project Type**: Web application (frontend + backend)

**Performance Goals**:
- API response time: <200ms p95 for CRUD operations
- Frontend load time: <3 seconds initial page load
- Support: 100 concurrent authenticated users

**Constraints**:
- API response time <200ms (p95)
- Data isolation 100% enforced (users never see others' tasks)
- All API endpoints require JWT authentication
- HTTPS in production (HTTP acceptable for local development)
- Step 1 console app must remain functional (no changes to backend/console/)

**Scale/Scope**:
- Users: 100 concurrent users minimum
- Tasks: Unlimited per user
- Sessions: 7-day JWT token expiration

- All endpoints require `Authorization: Bearer <token>` header

**вњ… X. Database Persistence**
- Neon Serverless PostgreSQL configured
- SQLModel for ORM (SQLAlchemy + Pydantic)
- Database migrations managed (Alembic)
- Connection pooling via Neon
- Indexes on foreign keys (user_id)
- DATABASE_URL in environment variables

**вњ… XI. Frontend-Backend Separation**
- Next.js frontend (port 3000)
- FastAPI backend (port 8000)
- Communication via REST API only
- CORS configured for http://localhost:3000
- JWT stored securely (httpOnly cookie recommended)
- Frontend attaches token to all API requests

**вњ… XII. JWT Authentication Flow**
- Better Auth configured with JWT plugin
- Shared BETTER_AUTH_SECRET between frontend and backend
- JWT verification middleware in FastAPI
- Token expiration: 7 days
- Invalid/expired tokens return 401 Unauthorized
- User ID extracted from JWT and validated against URL

**вњ… XIII. Responsive UI Design**
- Tailwind CSS for responsive layouts
- Mobile-first design approach
- Forms with validation and error feedback
- Loading states during API calls
- Success/error messages displayed to users
- Matches/exceeds console app functionality
